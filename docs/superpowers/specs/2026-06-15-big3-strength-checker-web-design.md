# BIG3 筋力レベルチェッカー（Web版 / React）— 設計書

- 作成日: 2026-06-15
- ステータス: ブレスト完了、ユーザーレビュー待ち
- 前身: Flutter版（`../iphoneApp`）。本プロジェクトはそれをWeb前提でReact/TypeScriptに再構築するもの。
- 引き継ぎ元参照: `docs/reference/flutter-version-spec.md`、`docs/reference/strength_standards.real.json`

## 1. 目的とゴール

- **第一目的（主）**: モダンなWebフロントエンド開発（React / TypeScript / Vite / Tailwind / Vitest）を一通り学ぶ。デプロイ（Vercel）とPWA化まで体験する。
- **第二目的（従）**: 実際にURLで配布し、自分や知人が使える状態にする。

アプリの価値: 性別と体重(kg)を入力すると、BIG3（ベンチプレス／スクワット／デッドリフト）について「自分の体重なら各レベルでどれくらい挙げれば良いか」が一目で分かる。データは内蔵、オフライン動作、サーバー不要。

## 2. スコープ

### やること（v1）
- 起動直後に筋力基準表を表示。
- 性別（男性／女性）と体重(kg)の入力・ローカル保存。
- BIG3の種目切り替え（ベンチ／スクワット／デッド）。
- 入力体重に対する各レベル基準値（初心者／初級／中級／上級／エリート）の表示（基準表＋1kg精度サマリー）。
- PWA対応（ホーム画面追加・オフライン）。
- Vercelで公開。

### やらないこと（YAGNI / 非ゴール）
- MAX重量を入力してレベルを自動判定する機能（v1は基準表方式。ただし後から足せる構造にする）。
- 年齢帯の切り替え（18-39のみ。データモデルに`ageBand`枠だけ用意し、将来40-49等を追加可能に）。
- World Recordのアプリ内表示（データには保持。将来表示に拡張可能）。
- ユーザーアカウント、クラウド同期、履歴保存。
- 多言語UIの実提供（v1は日本語のみ。i18n基盤も今は作らない＝YAGNI）。
- Docker（このプロジェクトには含めない。Dockerの価値を体感する別プロジェクトで扱う）。

## 3. 技術スタック

- **Vite + React + TypeScript**（モダンな標準構成）。
- **Tailwind CSS**（スタイリング）。
- **状態管理は React 標準のみ**（`useState` + localStorage）。ライブラリ（Zustand等）はこの規模では不要。
- **Vitest + React Testing Library**（テスト。ロジックはTDDで先にテスト）。
- **vite-plugin-pwa**（manifest + Service Worker 生成）。
- **Vercel**（静的ホスティングで公開）。
- 単位はkgネイティブ。lb→kg換算は不要（データがkgで提供されているため）。

## 4. 機能仕様

### 4.1 画面構成（モバイル優先・下部バー：A案）

単一画面 + 設定シート。画面下部に操作系を集約し、片手・親指で操作しやすくする。

```
┌─────────────────────────────┐
│  BIG3 レベルチェッカー   ベンチ・男性 │  ← 上部バー: タイトル＋現在の種目/性別
├─────────────────────────────┤
│  あなた(70kg)の目安 (kg)          │  ← サマリーカード（設定済みのとき表示）
│  初心者51.9 初級66.9 中級80.3    │     体重に対する各レベルの1kg精度補間値
│  上級110.3 エリート137.2          │
├─────────────────────────────┤
│  体重 初心 初級 中級 上級 ｴﾘｰﾄ    │  ← 体重別 基準表（スクロール可）
│  …                              │
│ ▶ 71  52  67  80.5 110.5 137.5  │  ← 設定体重に最も近い行をハイライト
│  …                              │
├─────────────────────────────┤
│ [ベンチ][スクワット][デッド] [⚙]    │  ← 画面下部バー
└─────────────────────────────┘
```

- 下部バー: ベンチ／スクワット／デッド（押すと表を即切替）＋ ⚙設定。
- ⚙設定 → 下からモーダルシート（性別[男性/女性]・体重[ kg ]・[保存]）。
- 保存すると表に戻り、該当行ハイライト＋サマリーカード表示。設定はlocalStorageに保存し次回復元。
- 初回（未設定）時はハイライト無しで全表を表示し「⚙設定から性別・体重を入力するとレベルが分かります」と案内。起動直後から表は見える。
- レスポンシブ: モバイル幅を基準に、PC幅では中央寄せ＋最大幅で読みやすく。

### 4.2 レベル区分

ExRx Strength Standards に準拠した5段階:
初心者(Untrained) / 初級(Novice) / 中級(Intermediate) / 上級(Advanced) / エリート(Elite)

## 5. データモデルとデータソース

### 5.1 データソース
- 出典: ExRx Strength Standards（kgページ）。
  - Squat: https://exrx.net/Testing/WeightLifting/SquatStandardsKg
  - Bench: https://exrx.net/Testing/WeightLifting/BenchStandardsKg
  - Deadlift: https://exrx.net/Testing/WeightLifting/DeadliftStandardsKg
- 年齢帯 18-39。男女 × BIG3 で計6テーブル。各テーブルは体重ブラケット（アンカー点）× 5レベル + World Record。
- 値はkgそのまま（換算不要）。`docs/reference/strength_standards.real.json` に取込済み。実装時に `src/data/strength_standards.json` へ移植。
- 公開時は出典「ExRx」を明記する。

### 5.2 データ構造の要点
- 男女で体重ブラケットが異なる（男: 52〜145+ / 女: 44〜90+）。テーブルごとに別アンカー配列で保持するため問題なし。
- `145+` / `90+` は上限オープン行（「それ以上」）。`plus: true` フラグで保持。
- `ageBand`（現状 "18-39" 固定）と `worldRecord`（保持・非表示）をデータに含め、将来拡張に備える。

### 5.3 1kg精度の算出（線形補間）
- 任意体重に対し、前後のアンカー点から各レベル値を直線補間。
  - 例: 体重67kgで中級77.5、75kgで中級85のとき、70kgの中級 = 77.5 + (85−77.5) × (70−67)/(75−67) = 80.3。
- 補間アンカーは閉じた行（男 52〜145 / 女 44〜90）を使用。
- 範囲外:
  - 最小ブラケット未満 → 最小行でクランプ＋「範囲外」注記。
  - 上限超過（145超 / 90超）→ `plus`行（145+ / 90+）の値を使用＋「範囲外」注記。
- サマリーカードはこの補間で1kg精度表示。

### 5.4 表の刻み（表示用の行生成）
- 100kg以下: 元アンカーに加え、隣接2点の中間体重（平均）の行を補間挿入し解像度を上げる。
- 100kg超: 元アンカーのみ表示。
- `plus`行（145+ / 90+）は「(上限)超」として末尾に表示。
- サマリーカードは表の刻みと無関係に常にユーザー体重で補間。

### 5.5 精度に関する注記
- ExRxの基準自体が目安であり、補間はその間を滑らかにつなぐ近似。実用上十分だが厳密な科学値ではない。将来 体重^(2/3) 等の曲線補間へ差し替え可能な構造にしておく。

## 6. アーキテクチャ

### 6.1 プロジェクト構成
```
src/
  main.tsx                    // エントリ
  App.tsx                     // 状態 {gender, exercise, bodyweight} を保持
  index.css                   // Tailwind ディレクティブ
  data/
    strength_standards.json   // ExRx実データ（kg）
    standards.ts              // JSONを型付きでロード
  domain/                     // ★純粋ロジック（UI非依存・テスト対象）
    types.ts                  // Gender / Exercise / Level / モデル型
    interpolation.ts          // 線形補間・クランプ・plus行処理
    displayRows.ts            // 表示用の行生成（≤100kgで中間行挿入）
    format.ts                 // kg表示フォーマット（小数第1位）
    level.ts                  // 【将来】MAX→レベル判定（v1は未実装の拡張点）
  state/
    useSettings.ts            // localStorage連動（性別・体重）
  components/
    Header.tsx                // 上部バー
    SummaryCard.tsx           // 体重別1kg精度サマリー
    StandardsTable.tsx        // 基準表＋該当行ハイライト
    ExerciseBottomBar.tsx     // 下部バー（種目＋⚙）
    SettingsSheet.tsx         // 性別・体重入力シート
public/
  (PWAアイコン・manifest関連)
```

### 6.2 状態管理
- `App.tsx` が `{gender, exercise, bodyweight}` を `useState` で保持し、propsで各コンポーネントへ。
- 性別・体重は `useSettings`（localStorage）で永続化・復元。
- 将来状態が増えたら Context / 軽量ライブラリ導入を検討（v1では不要）。

### 6.3 データフロー
1. 起動時に `strength_standards.json` を読み込み（importでバンドル）。
2. localStorageから設定（性別・体重）を復元。
3. 選択中の性別・種目で表データを生成（≤100kgは中間行挿入）し描画。
4. 体重が設定済みなら、その体重で各レベルを補間しサマリーカード表示、表の最も近い行をハイライト。
5. 下部バーで種目切替／⚙で性別・体重変更 → 状態更新 → 再描画。

## 7. エラー処理・バリデーション
- 体重入力は数値かつ妥当範囲（例: 30〜250kg）を検証し、不正値はインライン通知。
- 表の範囲外体重は端の値（または`plus`行）でクランプし「範囲外」注記。
- データ読込はビルド時importのため実行時の失敗は基本発生しない。

## 8. テスト方針（ロジックはTDD）
- 単体テスト（Vitest）: 線形補間（70kg→中級80.3の例）、最小未満・上限超過のクランプ、`plus`行の扱い、≤100kgの中間行生成、kgフォーマット。
- データテスト: JSONの読込・件数（6テーブル）・構造・各テーブルのアンカー昇順検証。
- コンポーネントテスト（React Testing Library）: 初回案内表示、設定反映時の行ハイライトとサマリー表示、種目切替。

## 9. PWA・公開
- `vite-plugin-pwa` で manifest（名称・アイコン・テーマ色・`display: standalone`）と Service Worker を生成。
- iOS Safariはホーム画面追加が手動（共有→ホーム画面に追加）。その案内を一言用意。
- Vercelに接続し、push時に自動デプロイ。`vite build` の静的成果物を配信。

## 10. 拡張ポイント（将来）
- **MAX自動判定**: `domain/level.ts` に「補間で出した各レベル基準値とユーザーのMAXを比較し、該当レベル＋次レベルまでの差を返す」関数を追加し、入力欄＋結果表示コンポーネントを足すだけ。
- **年齢帯**: データに他年齢帯テーブルを追加し、設定に年齢セレクタを足す。
- **World Record表示**: 保持済みの`worldRecord`を表/サマリーに表示。
- **多言語**: i18nライブラリ導入＋文言の外部化。
- **曲線補間**: 5.5の通り補間関数を差し替え可能に。
