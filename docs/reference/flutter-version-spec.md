# BIG3 筋力レベルチェッカー — 設計書

- 作成日: 2026-06-11
- ステータス: 承認済み（実装計画フェーズへ）

## 1. 目的とゴール

iPhoneアプリをApp Storeにリリースする一連の流れ（開発・署名・審査・公開）を実際に体験することが第一の目的。そのための題材として、機能を意図的にシンプルに保った筋力レベル確認アプリを作る。

アプリの価値: 性別と体重を入力すると、BIG3（ベンチプレス／スクワット／デッドリフト）について「自分の体重なら各レベルでどれくらい挙げれば良いか」が一目で分かる。データはアプリ内蔵で、オフラインで動作し、サーバーを必要としない。

## 2. スコープ

### やること（v1）
- 起動直後に筋力基準表を表示。
- 性別（男性／女性）と体重(kg)の入力。
- BIG3の種目切り替え（ベンチ／スクワット／デッドリフト）。
- 入力した体重に対する各レベル基準値（初心者／初級／中級／上級／エリート）の表示。
- 設定（性別・体重）のローカル保存。

### やらないこと（YAGNI / 非ゴール）
- ユーザーアカウント、クラウド同期、ログイン。
- トレーニング履歴の記録・保存。
- MAX重量を入力してレベルを自動判定する機能（v1は「基準表の該当行を見て自分で当てはめる」方式を採用）。
- 英語など多言語UIの実提供（仕組みだけ用意し、実翻訳は後続）。
- Android版のビルド・公開（Flutter採用で将来の道は確保するが、v1はiOSのみ）。

## 3. 技術スタック

- **Flutter（Dart）** を採用。
  - 採用理由: 将来の英語UI・グローバル展開・Android併用を見据えると、単一コードベースで両OS・多言語に対応でき、学ぶ言語が1つで済む。iOSリリースの流れ（証明書・プロビジョニング・App Store Connect・TestFlight・審査）はFlutterでも一通り体験できる。
  - 2026年6月時点でもFlutterはクロスプラットフォーム分野で主要かつ現役の選択肢であることを確認済み。
- 依存パッケージは最小限: `shared_preferences`（設定の永続化）、`flutter_localizations` + `intl`（多言語の土台）。
- 重い状態管理ライブラリ（Riverpod等）はv1では導入しない。

## 4. 機能仕様

### 4.1 画面構成（単一画面 + 設定シート）

画面下部に操作系を集約し、片手・親指で操作しやすくする。

```
┌─────────────────────────────┐
│  BIG3 レベルチェッカー  (ベンチ・男性)│  ← 上部バー: タイトル＋現在の種目/性別
├─────────────────────────────┤
│  あなた(70kg)の目安               │  ← サマリーカード（設定済みのとき表示）
│  初心者40 初級55 中級75          │
│  上級100 エリート125 (kg)        │
├─────────────────────────────┤
│  体重別 基準表（ベンチ・男性）      │
│  体重 初心者 初級 中級 上級 ELITE │
│  …                              │  ← スクロール可能
│ ▶ 70kg 40  55  75  100  125     │  ← 設定の体重に最も近い行をハイライト
│  …                              │
├─────────────────────────────┤
│ [ベンチ][スクワット][デッド] [⚙設定] │  ← 画面下部バー
└─────────────────────────────┘
```

- 下部バーは4要素: ベンチ／スクワット／デッド（押すと表を即切替）＋ ⚙設定。
- ⚙設定をタップすると、下からモーダルシートが出る:
  ```
  性別:  [ 男性 ] [ 女性 ]
  体重:  [  70  ] kg
            [ 保存 ]
  ```
- 保存すると表に戻り、該当行をハイライトしサマリーカードを表示。
- 設定はローカルに保存し、次回起動時に復元する。
- 初回起動（未設定）時は、ハイライト無しで全表を表示し「⚙設定から性別・体重を入力するとレベルが分かります」と案内。起動直後から表は見える。

### 4.2 レベル区分

ExRx Strength Standards に準拠した5段階:
初心者(Untrained) / 初級(Novice) / 中級(Intermediate) / 上級(Advanced) / エリート(Elite)

## 5. データモデルとデータソース

### 5.1 データソース
- 出典: ExRx Strength Standards（https://exrx.net/Testing/WeightLifting/StrengthStandards ）。
- 男女 × BIG3 で計6セット。各セットは複数の体重ブラケット（アンカー点） × 5レベルの1RM基準値。
- 原データはポンド(lb)。kgに換算して内蔵（表示・入力ともkg統一）。
- `assets/strength_standards.json` としてバンドルし、オフラインで利用。

### 5.2 1kg精度の算出（線形補間）
- 任意の体重に対し、前後のアンカー点から各レベル値を直線補間して算出する。
  - 例: 体重67kgで中級70kg、75kgで中級78kg のとき、70kgの中級 = 70 + (78−70) × (70−67)/(75−67) = 73kg。
- 範囲外（最小ブラケット未満／最大超過）は端の値でクランプし「範囲外」と注記。
- ユーザーの体重ピッタリの値はこの補間により1kg精度でサマリーカードに表示する。

### 5.3 表の刻み（表示用の行生成）
- フル基準表の行は次のルールで生成する:
  - 100kg以下: 元のアンカー点に加え、隣り合う2点の中間体重（平均）の行を補間して挿入し、解像度を上げる。
  - 100kg超: 元のアンカー点のみを表示。
- サマリーカードは表の刻みに関係なく、常にユーザーの体重で1kg精度の補間値を表示する。

### 5.4 精度に関する注記
- ExRxの基準自体が目安であり、補間はその間を滑らかにつなぐ近似である。実用上は十分だが厳密な科学値ではない。将来、体重^(2/3)等の曲線補間に差し替え可能な構造にしておく。

## 6. アーキテクチャ

### 6.1 プロジェクト構成
```
lib/
  main.dart                      // 起動・MaterialApp・テーマ・多言語設定
  models/strength_standard.dart  // 種目/性別/アンカー点/5レベルのモデル
  data/standards_repository.dart // JSON読込＋表データ提供
  logic/interpolation.dart       // 線形補間・クランプ・100kg以下の平均行生成
  services/settings_store.dart   // shared_preferencesで設定の保存/読込
  screens/home_screen.dart       // 上部バー＋サマリー＋表＋下部バー
  screens/settings_sheet.dart    // 性別・体重入力シート
  widgets/summary_card.dart
  widgets/standards_table.dart
  widgets/exercise_bottom_bar.dart
assets/strength_standards.json   // アンカーデータ（男女×BIG3、kg換算済）
test/
  interpolation_test.dart
  standards_repository_test.dart
  widget_test.dart
```

### 6.2 状態管理
- 単一画面のため最小構成。`HomeScreen` が `{性別, 体重, 選択中の種目}` を保持し `setState` で再描画する。
- 設定（性別・体重）は `shared_preferences` で永続化。
- 将来状態が増えたら Provider/Riverpod の導入を検討（v1では不要）。

### 6.3 データフロー
1. 起動時に `strength_standards.json` を1度読み込み、メモリ上に保持。
2. 保存済み設定（性別・体重）を `shared_preferences` から復元。
3. 選択中の性別・種目で表データを生成（100kg以下は平均行を挿入）し描画。
4. 体重が設定済みなら、その体重で各レベルを補間しサマリーカードに表示、表の最も近い行をハイライト。
5. 下部バーで種目切替／⚙設定で性別・体重変更 → 状態更新 → 再描画。

## 7. エラー処理
- 体重入力は数値かつ妥当な範囲（例: 30〜250kg）を検証し、不正値はインラインで通知。
- 表の範囲外の体重は端の値でクランプし「範囲外」と注記。
- JSON読込失敗時はエラー状態を表示（内蔵データのため通常は発生しない）。

## 8. 多言語対応（将来のグローバル展開準備）
- Flutter標準の `gen-l10n` ＋ ARBファイルで仕組みを最初から用意。
- v1は日本語（`app_ja.arb`）のみで出荷。英語は後日 `app_en.arb` を追加するだけで対応でき、コード変更はほぼ不要。
- すべてのユーザー向け文言はローカライズ経由にする。数値・単位(kg)はv1ではロケール非依存。

## 9. テスト方針（TDDで実装）
- 単体テスト: 線形補間（例の 70kg→73kg）、クランプ、100kg以下の平均行生成、lb→kg換算。
- リポジトリテスト: JSONの読込・件数・構造の検証。
- ウィジェットテスト: ホーム画面の描画、設定反映時の行ハイライトとサマリー表示。

## 10. リリース準備（本プロジェクトの主目的）
- Apple Developer Program 登録（年間 $99）が公開に必須。
- Xcode（最新）・CocoaPods を用意。`flutter build ipa` でビルドし、Transporter/Xcode 経由でアップロード。
- App Store Connect でアプリ登録、アイコン・スクリーンショット・プライバシー表示（本アプリはデータ収集なし＝「Data Not Collected」）、TestFlight、審査提出。
- バンドルID・署名（Xcodeの自動署名）・起動画面（Launch Screen）を整備。
- Androidはv1対象外。将来 `flutter build appbundle` で展開可能。

## 11. CI / Docker（練習トラック・スペック拡張）
- 目的: Dockerコンテナの実践練習。iOSビルド・署名はmacOS/Xcode必須でコンテナ化できないため対象外。
- 内容: GitHub Actions で Flutter公式系Dockerイメージ（`ghcr.io/cirruslabs/flutter:stable`）をコンテナとして使い、push/PR時に `flutter pub get` / `flutter analyze` / `flutter test` を実行する。
- ローカルでも同イメージで `docker run` してテストを回し、コンテナ操作を体験する。
- 将来のAndroid展開時は、同コンテナで `flutter build appbundle` を再現性ビルドに拡張できる。
